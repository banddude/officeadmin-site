Communications

Email Triage
Watches Gmail, classifies messages, drafts replies
▾

The comms pipeline scanner checks Gmail on a schedule. Morning scan at 7:45 AM, afternoon watchers at 3:00-3:45 PM. Each message is classified into one of three tiers.

Tier 1 messages are routine and predictable. These get auto-drafted and auto-sent at 9:30 AM without Mike touching them. Tier 2 messages get a notification. Tier 3 messages need human review and get queued as Apple Reminders.

Scanner → Classifier → Dispatcher → Drafter (x5) → Approval Queue

Python Gmail API Claude Scheduled

Drafter
Writes in Mike's voice using learned rules
▾

The Drafter reads entity context, routing rules, and Mike's writing conventions before composing any message. It produces emails, iMessages, and other written communication that sounds like Mike wrote them.

There is a learning pipeline. When Mike corrects a draft, the correction is distilled into a rule and added to the drafting conventions file. Over time, the drafts get better without retraining anything.

Entity context + ROUTING.md + DRAFTING.md → Claude → Draft → Mike's corrections → Updated rules

Claude Rule learning

iMessage
Two-way messaging with AIVA from Mike's phone
▾

Reads the native macOS Messages database (chat.db) via SQLite for search and history. Sends messages through BlueBubbles, a cross-platform relay that acts as a channel provider.

Mike can message AIVA from his phone like texting a person. AIVA can also send proactive notifications. Access is controlled by an allowlist with pairing approval, so not just anyone can message the system.

Python SQLite BlueBubbles

Notify
Notification routing across all devices
▾

A Flask API server running as a daemon on the Mac Mini. It provides an HTTP interface for sending notifications via iMessage, Talk (voice), or agent sessions. Other modules call it when they need to get Mike's attention.

Python Flask Daemon

Talk / Canvas
Voice and visual output on Mike's devices
▾

Talk pushes text-to-speech notifications to all of Mike's devices simultaneously using Apple Push Notification Service. AIVA can literally speak out loud through connected speakers.

Canvas is a phone-based HTML UI. AIVA can push interactive prompts, confirmation dialogs, and simple forms to Mike's phone screen. This is how AIVA shows information that does not fit in a text message.

APNS TTS HTML UI

Knowledge and Memory

MemPalace
SQLite knowledge graph with thousands of entities
▾

A SQLite-backed knowledge graph organized into rooms and drawers. Stores entities, relationships (triples), and facts extracted from conversations, documents, and agent sessions.

Supports graph traversal, search, timeline queries, and tunnel discovery (finding non-obvious connections between concepts). Uses priority-based ingest ordering: agent sessions first, then memory, cron runs, Claude projects, and Codex artifacts.

mempalace search "Shaffer Construction" mempalace kg-query "who works for" mempalace traverse entity-123 --depth 3

Python SQLite CLI

Voice Pipeline
Transcription and speaker identification from recordings
▾

iPhone recordings sync via iCloud to the Mac Mini. A launchd watcher detects new files and triggers the transcription pipeline.

AssemblyAI handles speech-to-text with speaker diarization. Pinecone stores voice embeddings so the system can identify who is speaking across all recordings, even in new conversations. Transcripts feed into MemPalace for knowledge extraction and get linked to the relevant entities.

iPhone → iCloud sync → launchd watcher → AssemblyAI → Pinecone speaker ID → SQLite → MemPalace

Python AssemblyAI Pinecone launchd

CRM / Entity System
GitHub-backed relationship management
▾

The banddude/mikeshaffer GitHub repo is the operational backbone. Every customer, vendor, crew member, and family member gets an entity folder with a JSON profile, task files, and work records in Markdown.

Work lifecycle tracks from lead to payment: lead, bidding, bid, negotiating, awarded, scheduled, in_progress, complete, invoiced, paid. GitHub Actions run validators on every push to keep the data clean.

entities/ acme-corp/ entity.json # profile data tasks/ # task files work/ # job records, scope docs notes.md # free-form notes

GitHub JSON GitHub Actions

Business Operations

QuickBooks CLI
Programmatic access to accounting and invoicing
▾

A Python and Node CLI with about 40 files that wraps the QuickBooks Online REST API. OAuth credentials stored in macOS Keychain. Creates and manages estimates, invoices, bills, customers, and vendors without opening the QuickBooks app.

Also integrates with QuickBooks Time (formerly TSheets) for crew scheduling and timesheets. Contact records in Apple Contacts link to QuickBooks customer profiles via URL labels.

Python Node.js OAuth REST API

Apple Ecosystem

Contacts
Direct SQLite access to Apple Contacts
▾

Reads and writes to the macOS Contacts database directly via SQLite, bypassing AppleScript for speed. Every person in the system has a Contacts entry that links to their entity folder in the CRM repo via URL labels.

This is the identity bridge. When a phone number comes in through iMessage or a name appears in an email, the contacts module resolves it to a canonical entity.

Python SQLite PyObjC

Reminders
Apple Reminders as the universal approval queue
▾

Apple Reminders is the approval queue for the entire system. Every draft email, generated estimate, and identified task becomes a Reminder. Mike reviews and approves from his phone, iPad, or Mac.

Reminders sync via iCloud, so they show up everywhere. Tasks also mirror to entity folders in the GitHub CRM. The subtask tool allows breaking complex reminders into actionable steps.

EventKit iCloud Sync

Infrastructure

Agent Runtime
Background agent sessions in tmux
▾

Background agent sessions run in tmux with per-agent workspaces and runtime configs. The agents CLI manages session lifecycle, the agent-revive CLI restarts crashed sessions, and the tell CLI sends commands to running sessions.

This is how AIVA handles tasks that take minutes or hours without blocking the main conversation. Long builds, research tasks, and multi-step workflows all run as background agents.

Bash tmux
