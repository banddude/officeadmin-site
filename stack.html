Stack

Everything runs on a Mac Mini. No Kubernetes, no containers, no cloud compute. Just Python, SQLite, and tmux.

The layers

The system is built in five layers. The .aiva/ layer is the custom-built part. Everything else is open-source tooling, Apple frameworks, or third-party APIs.

1. Hardware
Mac Mini (Intel i9) at aiva.local, running macOS
▾

A Mac Mini sits at home, always on, connected to everything via Tailscale mesh VPN. Cloudflare Tunnels handle secure ingress without port forwarding. Phones, laptops, and speakers throughout the house are all participants on the same network.

2. System services
macOS daemons, tunnels, and session management
▾

launchd tmux nginx cloudflared Tailscale macOS Keychain

launchd supervises all daemons and scheduled jobs. tmux manages background agent sessions. nginx handles local reverse proxy for OAuth callbacks. cloudflared provides secure tunnels to Cloudflare. All credentials live in macOS Keychain.

3. Open-source tools
Third-party open-source software the system uses
▾

OpenClaw MemPalace BlueBubbles Akaunting Home Assistant Playwright

OpenClaw is the agent orchestration layer. It schedules 83 cron jobs, routes messages across channels (iMessage, Google Chat), hosts the MCP gateway that exposes AIVA's skills, runs subagents in isolated sessions, and maintains its own short-term memory cache with FTS and vector search. MemPalace is an open-source SQLite knowledge graph tool for long-term temporal memory. BlueBubbles is the cross-platform iMessage bridge. Akaunting is the open-source business hub.

4. .aiva/ — The custom layer
The custom infrastructure that ties everything together
▾

This is the custom-built layer. The ~/.aiva/ directory contains all the custom modules, skills, scripts, MCP servers, state management, and orchestration logic that make the system work as a whole.

It includes the email triage pipeline, the Drafter (voice-matching composition with learning rules), the voice memo transcription pipeline, the contacts/QuickBooks/Google CLIs, the Talk CLI and Canvas phone UI, the notify API server, the agent session manager, the daily sync scripts, and all the glue that connects open-source tools to Apple frameworks to third-party APIs.

Python 3 Bash Node.js 50+ modules MCP servers

5. Agent brains
The LLMs that power reasoning and extraction
▾

Claude (Anthropic) is the primary reasoning engine. AIVA runs on Claude Code with MCP servers providing tool access to all modules. Claude handles email classification, draft composition, conversation understanding, and general task execution.

GLM-5.1 handles batch knowledge extraction for MemPalace. It reads session logs from Claude, Codex, and OpenClaw, extracts entities and relationships, and writes them into the knowledge graph.

AssemblyAI provides speech-to-text with speaker diarization. Pinecone stores voice embeddings for speaker identification across recordings.

Why a Mac Mini?

Most of what AIVA does requires access to macOS-native APIs. Contacts, Reminders, Calendar, Notes, and Messages all use Apple frameworks (EventKit, PyObjC, SQLite databases in ~/Library). iCloud sync means changes on the Mac Mini show up on the operator's phone instantly. The Apple ecosystem is the integration layer. A Linux server would need to replicate all of that from scratch.

The Mac Mini sits at home, always on, connected via Tailscale. Cloudflare Tunnels handle secure ingress. There is no cloud compute bill for the core system. The only external costs are the third-party APIs (Claude, AssemblyAI, Pinecone, Google Workspace, QuickBooks).
